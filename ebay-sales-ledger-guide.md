# 売買管理台帳2026 使い方ガイド

**eBay で海外に商品を売るビジネスの「お金の記録」を、Excel で管理するためのツールの使い方ガイドです。**

---

## この台帳でできること

eBay（海外向けネットオークション）で商品を売ったときの記録を、Excel のマクロ（自動プログラム）でかんたんに管理できます。

おもにこんなことをやってくれます：

- eBay から書き出したデータを、きれいに整理してくれる
- 売った商品の **手数料・送料・関税**（海外に送るときにかかるお金）を自動で記録してくれる
- 国内の EC サイト（メルカリなど）で仕入れた商品と、eBay で売った商品を AI が自動で照合（つきあわせ）してくれる
- 確定申告に必要な **仕訳**（お金の出入りの記録）を自動で作ってくれる

---

## 用語の説明（さきに読んでね）

このガイドでは、ちょっとむずかしい言葉がたくさん出てきます。さきに意味を知っておくと、読みやすくなります。

| 言葉 | 意味 |
|------|------|
| 売買管理帳 | 売ったもの・買ったものをまとめて記録する表のこと |
| CSV | データをカンマで区切ったファイル。Excel で開ける |
| マクロ | Excel の中で動く自動プログラムのこと |
| 転記 | あるシートのデータを、別のシートにコピーすること |
| 照合 | 2つのデータを比べて、同じものを見つけること |
| 按分（あんぶん） | まとめてかかったお金を、商品ごとに分けること |
| 仕訳（しわけ） | 「何にいくら使った」「何でいくらもらった」を記録すること。確定申告に必要 |
| 為替レート | 外国のお金（ドル）を日本のお金（円）に変える計算のための数字 |
| 手数料 | eBay に払う利用料金のこと |
| 関税（かんぜい） | 海外に商品を送るとき、国にはらうお金のこと |
| 送料 | 商品を届けるためにかかるお金のこと |
| トラッキング番号 | 荷物の追跡番号。どの荷物がどこにあるかを調べるための番号 |

### グループ 1: メインの管理画面

| シート名 | なにをするシート？ |
|----------|------------------|
| ボタン | 操作ボタンが並んでいるシート。ここからすべての操作を行う |

| 利益計算 | もうけ（利益）を計算してくれるシート |

### グループ 2: eBay のデータ置き場

| シート名 | なにをするシート？ |
|----------|------------------|
| ebay_all | eBay からダウンロードした販売データを貼り付ける場所 |
| Trans_all | eBay の取引明細（お金のやりとり記録）を貼り付ける場所 |
| Order | 注文データが自動で振り分けられる場所 |
| Refund | 返品データが自動で振り分けられる場所 |
| Trans_集計 | 取引明細の集計結果が自動で作られる場所 |

### グループ 3: 送料・関税のデータ置き場

| シート名 | なにをするシート？ |
|----------|------------------|
| ORC_ALL | 送料（Orange 便）の生データを貼り付ける場所 |
| ORC_TTL | 送料の集計結果が自動で作られる場所 |
| CTM_ALL | 関税の生データを貼り付ける場所 |
| CTM_TTL | 関税の集計結果が自動で作られる場所 |

### グループ 4: 国内仕入れ

| シート名 | なにをするシート？ |
|----------|------------------|
| ECサイト | メルカリ・ヤフオクなどで買った（仕入れた）商品のデータ |

### グループ 5: 仕訳・設定

| シート名 | なにをするシート？ |
|----------|------------------|
| 売上 | 売上の仕訳データが自動で作られる場所 |
| 仕入 | 仕入れの仕訳データが自動で作られる場所 |
| その他費用 | eBay の利用料などの仕訳データが自動で作られる場所 |
| 適用為替レート | ドル→円の変換レートを月ごとに設定する場所 |
| 設定 | EC 照合のカテゴリー設定をする場所 |
| 仕訳設定 | 仕訳の科目（お金の分類名）を設定する場所 |

## 作業の全体の流れ

作業は大きく **5 つのフェーズ**（段階）に分かれています。上から順番にやっていきます。

```
フェーズ 1: eBay のデータを取り込む
    ↓
フェーズ 2: 手数料・送料・関税を照合する
    ↓
フェーズ 3: 国内で仕入れた商品を照合する
    ↓
フェーズ 4: 仕訳を作成する
    ↓
フェーズ 5: ファイルを書き出す
```

---

## フェーズ 1: eBay のデータを取り込む（ボタン 1〜2）

### やること

eBay の管理画面からダウンロードした CSV データを、売買管理帳に取り込みます。

### 手順

#### ステップ 1: CSV を貼り付ける

1. eBay の管理画面（Seller Hub）から **販売データの CSV** をダウンロードする
2. CSV ファイルを Excel で開く
3. 中身をぜんぶコピー（Ctrl + A → Ctrl + C）
4. 売買管理台帳の **「ebay_all」シート** に貼り付ける（Ctrl + V）

> 同じように、取引明細の CSV は **「Trans_all」シート** に貼り付けてください。

#### ステップ 2: ボタン 1「ebay_all データ整理」を押す

1. **「ボタン」シート** に移動する
2. **「ebay_all データ整理」** ボタンをクリック
3. 確認のメッセージが出るので **「はい」** をクリック
4. 「完了しました」と出ればOK

**このボタンがやってくれること：**
- 日付のデータを、Excel が理解できる形に変換する
- ドルの金額（`$12.50` のような文字）を、計算できる数字に変換する
- データを番号順にならべかえる
- 1 つの注文に複数の商品がある場合、金額を商品ごとに割りふる

#### ステップ 3: ボタン 2「ebay_all → 売買管理帳」を押す

1. **「ebay_all → 売買管理帳 コピー」** ボタンをクリック
2. 確認メッセージで **「はい」** をクリック
3. 「完了しました」と出ればOK

**このボタンがやってくれること：**
- ebay_all シートのデータを、売買管理帳に自動でコピーする
- ドル金額 → 円金額の変換（為替レートを使って自動計算）
- 売上合計や eBay 手数料の計算式を自動で入れてくれる

> **大事なこと：** 為替レートは **「適用為替レート」シート** にあらかじめ月ごとに入力しておいてください。

## フェーズ 2: 手数料・送料・関税を照合する（ボタン 3〜4、7〜10）

### やること

eBay に払った手数料、配送にかかった送料、海外に送るときの関税を、売買管理帳の各商品に記録します。

### 2-1: 手数料の照合（ボタン 3 → 4）

#### ステップ 1: ボタン 3「Trans_集計 作成」を押す

Trans_all（取引明細）シートのデータを、集計用のシートに整理します。

**このボタンがやってくれること：**
- Trans_all のデータを Trans_集計 シートにコピーする
- 金額の `--` を `0` に変換する
- ドル表示に統一する
- 返品（Refund）のデータに色をつけてわかりやすくする

#### ステップ 2: ボタン 4「Trans_集計 → 売買管理帳」を押す

**このボタンがやってくれること：**
- Trans_集計 の手数料データを、売買管理帳の **M列（手数料）** に自動で記録する
- 注文番号（Order ID）を使って、どの商品の手数料かを自動で見つけてくれる

### 2-2: 送料の照合（ボタン 7 → 8）

#### ステップ 1: 送料データを貼り付ける

1. 送料データ（Orange 便の明細）の CSV をダウンロードする
2. **「ORC_ALL」シート** に貼り付ける

#### ステップ 2: ボタン 7「送料ORC 集計」を押す

**このボタンがやってくれること：**
- 同じトラッキング番号（荷物の追跡番号）の送料をまとめて合計する
- 結果を ORC_TTL シートに出力する

#### ステップ 3: ボタン 8「送料ORC → 売買管理帳」を押す

**このボタンがやってくれること：**
- ORC_TTL の送料を、売買管理帳の **Q列（送料）** に自動で記録する
- トラッキング番号を使って、どの商品の送料かを自動で見つけてくれる
- 1 つの荷物に複数の商品が入っている場合、送料を商品ごとに **按分**（均等に分ける）してくれる
  - 例：送料 1,000円で商品が 3 つの場合 → 334円、333円、333円（合計がぴったり 1,000円になるように調整）

### 2-3: 関税の照合（ボタン 9 → 10）

送料とまったく同じ流れです。

#### ステップ 1: 関税データを貼り付ける

1. 関税データの CSV をダウンロードする
2. **「CTM_ALL」シート** に貼り付ける

#### ステップ 2: ボタン 9「関税CTM 集計」を押す

- 同じ注文番号の関税をまとめて合計して、CTM_TTL シートに出力する

#### ステップ 3: ボタン 10「関税CTM → 売買管理帳」を押す

- CTM_TTL の関税を、売買管理帳の **R列（関税）** に自動で記録する
- 送料と同じように、複数商品がある場合は按分してくれる

## フェーズ 3: 国内で仕入れた商品を照合する（ボタン 5）

### やること

メルカリやヤフオクなどで買った（仕入れた）商品が、eBay でどの商品として売れたかを、**AI（人工知能）が自動で判断** してくれます。

### 事前準備

#### OpenAI API Key の設定

この機能は OpenAI（ChatGPT を作っている会社）の AI を使います。使うには **API Key**（パスワードのようなもの）が必要です。

1. OpenAI のサイトで API Key を取得する
2. Excel のマクロ画面（Alt + F11）で `SetOpenAIApiKey` を実行する
3. 入力画面が出るので、API Key を貼り付けて OK をクリック

> API Key は一度設定すれば、ファイルに保存されるので毎回やる必要はありません。

#### EC サイトのデータを貼り付ける

1. 国内 EC サイトの購入履歴データを **「ECサイト」シート** に貼り付ける
   - メルカリ、ヤフオク、Yahoo!フリマ、ラクマ、その他 EC のデータをまとめて入れる

### 手順

#### ボタン 5「ECサイト照合 → 売買管理帳」を押す

1. ボタンをクリックすると、EC サイトの照合メニューが表示される
2. 個別にサイトを選ぶか、**「一括照合」** ですべてまとめてやるかを選ぶ

| 選択肢 | 対象 |
|--------|------|
| メルカリ 照合 | メルカリで買ったものだけ |
| ヤフフリ 照合 | Yahoo!フリマで買ったものだけ |
| ラクマ 照合 | ラクマで買ったものだけ |
| ヤフオク 照合 | ヤフオクで買ったものだけ |
| その他EC 照合 | その他のサイトで買ったものだけ |
| 一括照合 | ぜんぶまとめて照合（おすすめ） |

**このボタンがやってくれること：**
1. EC サイトで買った商品の名前を AI に送る
2. AI が売買管理帳の中から「これと同じ商品だ！」というものを探す
3. 見つかったら、売買管理帳に仕入先・仕入日・仕入価格などを自動で記録する
4. さらに、商品を 16 カテゴリー（ゲーム、トレカ、カメラなど）に自動分類してくれる
5. 仕入先ごとに色分けしてくれる：
   - メルカリ → ピンク
   - Yahoo!フリマ → 青
   - ラクマ → 緑
   - ヤフオク → オレンジ
   - その他 → 紫

> **照合できなかった商品** は「X」マークがつきます。手動で確認してください。

## フェーズ 4: 仕訳を作成する（ボタン 11〜14）

### 仕訳とは？

「仕訳」とは、お金の出入りを **「何のお金が」「いくら」動いたか** を記録することです。確定申告（税金の申告）をするために必要です。

たとえば：
- 「メルカリでゲームソフトを 3,000 円で買った」→ **仕入の仕訳**
- 「eBay でゲームソフトが $50 で売れた」→ **売上の仕訳**
- 「eBay から手数料 $5 を引かれた」→ **費用の仕訳**

この台帳では、4 種類の仕訳をボタンひとつで自動作成できます。

### 4-1: 仕入の仕訳（ボタン 11）

#### ボタン 11「仕入 仕訳出力」を押す

**このボタンがやってくれること：**
- EC サイトシートの仕入データをもとに、**「仕入」シート** に仕訳を自動で作成する
- 全サイト（メルカリ・ヤフフリ・ラクマ・ヤフオク・その他）を一括で出力する
- 各仕訳には、カテゴリー＋連番の **仕訳ナンバー** が自動でつく
  - 例：`VGC000000001`（ゲームカテゴリーの 1 番目）

**出力される内容（仕入シートに記録されること）：**

| 項目 | 内容 |
|------|------|
| 日付 | 仕入れた日 |
| 勘定科目 | 「仕入高」 |
| 金額 | 仕入れた金額 |
| 摘要 | 仕入先名 + 仕訳ナンバー |

### 4-2: 売上の仕訳（ボタン 12）

#### ボタン 12「売上 仕訳出力」を押す

**このボタンがやってくれること：**
- 売買管理帳のデータをもとに、**「売上」シート** に仕訳を自動で作成する
- 1 つの売上に対して、2 行の仕訳が作られる：
  - **売上金額**（商品が売れた金額）
  - **手数料**（eBay に払った手数料）
- ドル→円の変換も自動でやってくれる

### 4-3: 売上返品の仕訳（ボタン 13）

#### ボタン 13「売上返品 仕訳出力」を押す

返品（お客さんが商品を返してきた場合）があったときだけ使います。

**このボタンがやってくれること：**
- Trans_集計 の返品（Refund）データをもとに、**「売上」シート** に返品仕訳を作成する
- 売上を取り消す仕訳が自動で作られる

> 返品がない月は、このボタンは押さなくて OK です。

### 4-4: その他費用の仕訳（ボタン 14）

#### ボタン 14「その他費用 仕訳出力」を押す

**このボタンがやってくれること：**
- Trans_集計 の中から、eBay のストア利用料（月額料金）などを抽出する
- **「その他費用」シート** に仕訳を自動で作成する

---

## フェーズ 5: ファイルを書き出す（ボタン 6）

### ボタン 6「保存用 export」を押す

すべての作業が終わったら、このボタンで きれいな Excel ファイルを書き出します。

**このボタンがやってくれること：**
- ボタンシートや設定シートなど、提出に不要なシートを除外した `.xlsx` ファイルを作成する
- マクロを含まないファイルなので、他の人に安全に渡すことができる

## 困ったときは（トラブルシューティング）

### ボタンを押しても何も起きない

**やってみること：**
- [ ] マクロが有効になっているか確認する（ファイル → オプション → トラストセンター → マクロの設定 → 「すべてのマクロを有効にする」）
- [ ] ファイルを一度閉じて、もう一度開く
- [ ] 拡張子が `.xlsm`（マクロ有効ブック）になっているか確認する

### 「データがありません」と出る

**やってみること：**
- [ ] CSV データがちゃんと正しいシートに貼り付けられているか確認する
- [ ] 1 行目（ヘッダー行）が含まれているか確認する
- [ ] 前のボタンを先に実行しているか確認する（順番を守ってください）

### EC照合で「API Key が設定されていません」と出る

**やってみること：**
- [ ] OpenAI の API Key を設定する（Alt + F11 → `SetOpenAIApiKey` を実行）
- [ ] API Key が正しいか確認する（OpenAI のサイトで確認）

### 照合結果がおかしい（間違った商品にマッチしている）

**やってみること：**
- [ ] 売買管理帳の商品名と EC サイトの商品名を目で見くらべて、手動で修正する
- [ ] 「設定」シートでカテゴリー別の条件を追加・調整する

### 仕訳の金額がおかしい

**やってみること：**
- [ ] 「適用為替レート」シートの為替レートが正しいか確認する
- [ ] 「仕訳設定」シートの勘定科目が正しいか確認する
- [ ] 元データ（売買管理帳）の金額が正しいか確認する

### ボタンが消えてしまった

**やってみること：**
- Excel のマクロ画面（Alt + F11）を開いて、「イミディエイトウィンドウ」に以下を入力して Enter：
```
RestoreButtons
```
- これで 14 個のボタンがすべて復活します

## よくある質問

### Q. 作業の順番を間違えたらどうなる？
**A. やり直せます。** ただし、前のステップのデータがないとエラーになることがあります。できるだけチェックリストの順番通りにやってください。

### Q. 毎月やらないといけない？
**A. はい。** eBay の販売データは月ごとにダウンロードして、毎月この作業をします。ただ、ボタンを順番に押すだけなので、慣れれば 30 分くらいで終わります。

### Q. EC 照合に使う AI の料金はかかる？
**A. 少しかかります。** OpenAI API は従量課金（使った分だけお金がかかる）ですが、この用途では 1 回の照合で数円〜数十円程度です。

### Q. 確定申告のソフトにそのまま使える？
**A. 仕訳シートのデータを、弥生会計などの会計ソフトにインポートできる形式で出力しています。** そのまま取り込めるはずですが、会計ソフトの仕様に合わせて微調整が必要な場合もあります。

### Q. データのバックアップは必要？
**A. 必ずとってください。** マクロ実行前に、ファイルをコピーしてバックアップを作っておくと安心です。
