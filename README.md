# EC購入履歴 → Google Sheets 拡張機能ガイド

**ネットで買ったものの記録を、かんたんに Google スプレッドシートに保存できる Chrome 拡張機能の使い方ガイドです。**

---

## この拡張機能でできること

ネットショッピングで買ったものの情報（商品名・値段・買った日など）を、ボタンひとつで Google スプレッドシート（Excel のようなもの）に書き出してくれます。

手で一つずつ入力する必要がなくなるので、とっても便利です。

### 対応しているサイト（12サイト）

| サイト名 | どんなサイト？ |
|----------|--------------|
| メルカリ | フリマアプリ（中古品の売り買い） |
| ラクマ | フリマアプリ |
| Yahoo!フリマ | フリマアプリ |
| ヤフオク | ネットオークション |
| Yahoo!ショッピング | ネット通販 |
| Amazon | ネット通販 |
| ブックオフオンライン | 中古の本・ゲームなどのネット通販 |
| ネットオフ | 中古品のネット通販 |
| 駿河屋 | 中古のゲーム・本・フィギュアなどのネット通販 |
| ゲオオンライン | 中古ゲーム・DVDなどのネット通販 |
| リコレ（ソフマップ） | 中古の家電・パソコンなどのネット通販 |
| ふるいちオンライン | 中古品のネット通販 |

## セットアップの手順（全体の流れ）

セットアップは大きく分けて **3つのステップ** があります。

```
ステップ1: Google の設定をする（準備）
    ↓
ステップ2: 拡張機能を Chrome に入れる（インストール）
    ↓
ステップ3: 拡張機能と Google をつなげる（認証）
```

少しむずかしそうに見えますが、一つずつやれば大丈夫です！

---

## ステップ1: Google の設定をする

Google に「この拡張機能がスプレッドシートを使っていいよ」と教えてあげる作業です。

### 1-1. Google Cloud Console を開く

1. Chrome で **Google Cloud Console** を開きます
   - URL: `https://console.cloud.google.com/`
2. 自分の Google アカウントでログインします

### 1-2. 新しいプロジェクトを作る

「プロジェクト」とは、Google の中に作る「作業部屋」のようなものです。

1. 画面の上のほうにある **プロジェクトを選択** と書いてあるところをクリック
2. 右上の **「新しいプロジェクト」** をクリック
3. 以下のように入力します：
   - **プロジェクト名**: `purchase-history`（好きな名前でOK）
   - **場所**: 「組織なし」のままでOK
4. **「作成」** ボタンをクリック
5. 画面の上に、いま作ったプロジェクトの名前が表示されていれば OK

### 1-3. Google Sheets API をオンにする

「API」とは、プログラムどうしが会話するための仕組みです。ここでは「拡張機能がスプレッドシートを操作してもいいよ」という許可を出します。

1. 画面の左にあるメニューから **「API とサービス」** → **「ライブラリ」** を選ぶ
2. 検索バーに **`Google Sheets API`** と入力
3. 出てきた **「Google Sheets API」** をクリック
4. **「有効にする」** ボタンをクリック
5. 「API が有効です」と出れば OK

### 1-4. Google Drive API をオンにする（やっておくと便利）

スプレッドシートを好きなフォルダに保存したい場合に必要です。

1. 同じように **「API とサービス」** → **「ライブラリ」** を開く
2. 検索バーに **`Google Drive API`** と入力
3. 出てきた **「Google Drive API」** をクリック
4. **「有効にする」** ボタンをクリック

> **やらなくても動きます！** ただし、フォルダを選ぶ機能が使えなくなり、スプレッドシートは「マイドライブ」の一番上に作られます。

### 1-5. OAuth 同意画面を設定する

「OAuth（オーオース）」とは、「この拡張機能に自分のデータを使わせてもいいですよ」という許可の仕組みです。

#### 基本情報の入力

1. 左メニューから **「API とサービス」** → **「OAuth 同意画面」** を選ぶ
2. **「始める」** または **「同意画面を構成」** をクリック
3. User Type は **「外部」** を選んで **「作成」** をクリック
4. 以下を入力します：
   - **アプリ名**: `購入履歴 Sheets`（好きな名前でOK）
   - **ユーザーサポートメール**: 自分の Gmail を選ぶ
   - **デベロッパーの連絡先情報**: 自分のメールアドレスを入力
5. **「保存して次へ」** をクリック

#### スコープ（権限）の設定

「スコープ」とは、「何をしていいか」の範囲のことです。

1. **「スコープを追加または削除」** をクリック
2. 検索して、以下の2つにチェックを入れます：
   - `Google Sheets API` の **`.../auth/spreadsheets`**
   - `Google Drive API` の **`.../auth/drive.file`**
3. **「更新」** をクリック
4. **「保存して次へ」** をクリック

#### テストユーザーの追加

「テストユーザー」とは、この拡張機能を使える人のことです。自分を登録します。

1. **「+ ADD USERS」** をクリック
2. **自分の Gmail アドレス** を入力
3. **「追加」** → **「保存して次へ」** をクリック

> **大事なこと:** ここに登録した Google アカウントだけが拡張機能を使えます。ほかの人にも使ってもらいたい場合は、その人のメールアドレスも追加してください（最大100人まで）。

### 1-6. OAuth クライアント ID を作る

「クライアント ID」とは、拡張機能の身分証明書のようなものです。

1. 左メニューから **「API とサービス」** → **「認証情報」** を選ぶ
2. **「+ 認証情報を作成」** → **「OAuth クライアント ID」** をクリック
3. 以下を設定します：
   - **アプリケーションの種類**: **「Chrome 拡張機能」** を選ぶ
   - **名前**: `purchase-history`（好きな名前でOK）
   - **アイテム ID**: 拡張機能の ID を入力（→ ステップ2で確認できます。先にステップ2をやってもOK）
4. **「作成」** をクリック
5. 表示される **クライアント ID** をコピーしてメモしておく

> **拡張機能 ID の確認方法**は、次のステップ2で説明します。

## ステップ2: 拡張機能を Chrome に入れる

### 2-1. ファイルを準備する

1. 開発者からもらった **ZIP ファイル** をパソコンに保存
2. ZIP ファイルを **右クリック** → **「すべて展開」** で中身を取り出す

### 2-2. Chrome に読み込む

1. Chrome のアドレスバーに **`chrome://extensions/`** と入力して Enter
2. 画面の右上にある **「デベロッパーモード」** のスイッチをオンにする
3. **「パッケージ化されていない拡張機能を読み込む」** ボタンをクリック
4. さっき展開したフォルダを選ぶ
5. 拡張機能が追加されて、Chrome のツールバー（画面の右上あたり）にアイコンが出れば OK

### 2-3. 拡張機能 ID を確認する

`chrome://extensions/` の画面に、拡張機能の **ID**（英語の小文字と数字の長い文字列）が表示されています。これをメモしてください。

> この拡張機能は ID が固定されているので、通常は **`kpjkejoknpjhmbgdpiodfcldhkgcejbj`** と表示されます。もし違う ID が表示されたら、そちらを使ってください。

**ステップ1-6 をまだやっていない場合は、ここでメモした ID を使って 1-6 を完了してください。**

### 2-4. manifest.json の確認（ふつうはやらなくてOK）

拡張機能のフォルダの中にある `manifest.json` というファイルに、クライアント ID が書かれています。

ステップ1-6 で作ったクライアント ID と、ファイルに書かれている ID が **違う場合だけ**、ファイルを開いて書き換えてください。

```json
{
  "oauth2": {
    "client_id": "ここにステップ1-6でコピーしたクライアントIDを貼る",
    "scopes": [
      "https://www.googleapis.com/auth/spreadsheets",
      "https://www.googleapis.com/auth/drive.file"
    ]
  }
}
```

同じなら、何もしなくて大丈夫です。

## ステップ3: 拡張機能と Google をつなげる（認証）

### 3-1. 認証する

1. Chrome の右上にある拡張機能のアイコンをクリック（ポップアップ画面が開きます）
2. **「Google 認証」** ボタンをクリック
3. Google アカウントの選択画面が出るので、自分のアカウントを選ぶ
4. **「このアプリは Google で確認されていません」** という警告が出ることがあります。こわくないので大丈夫！
   - **「詳細」** をクリック
   - **「〇〇（安全ではないページ）に移動」** をクリック
5. 「スプレッドシートの操作を許可しますか？」のような画面が出るので、**「許可」** をクリック
6. ポップアップに戻って、ステータスが **緑色（認証済み）** になったら成功！

## 使い方

セットアップが終わったら、あとはかんたんです！

### 基本の使い方

1. 買い物をしたサイトの **購入履歴ページ** を Chrome で開く
2. 拡張機能のアイコンをクリック
3. **「抽出開始」** ボタンをクリック
4. 自動でスプレッドシートが作られて、購入履歴が書き込まれます

### 各サイトの購入履歴ページの開き方

| サイト | 購入履歴ページの開き方 |
|--------|---------------------|
| メルカリ | マイページ → 購入した商品 |
| ラクマ | マイページ → 購入した商品 |
| Yahoo!フリマ | マイページ → 購入した商品 |
| ヤフオク | マイオク → 落札分 |
| Yahoo!ショッピング | 注文履歴 |
| Amazon | 注文履歴 |
| ブックオフオンライン | マイページ → 注文履歴 |
| ネットオフ | マイページ → 注文履歴 |
| 駿河屋 | マイページ → 注文履歴 |
| ゲオオンライン | マイページ → 注文履歴 |
| リコレ（ソフマップ） | マイページ → 注文履歴 |
| ふるいちオンライン | マイページ → 注文履歴 |

### こんなこともできます

#### スプレッドシートを指定する

毎回新しいスプレッドシートを作るのではなく、1つのスプレッドシートにまとめたい場合：

1. ポップアップの **「スプレッドシートID」** の欄に、使いたいスプレッドシートの ID を入力
   - スプレッドシートの URL の `https://docs.google.com/spreadsheets/d/`**ここの部分**`/edit` が ID です
2. これで、毎回同じスプレッドシートに追記されます

#### 保存先フォルダを指定する

スプレッドシートを Google ドライブの特定のフォルダに保存したい場合：

1. ポップアップの **「保存先フォルダ」** の欄に、Google ドライブのフォルダ URL を貼り付け
2. 新しく作られるスプレッドシートが、そのフォルダに保存されます

#### 抽出する期間を選ぶ

全部の購入履歴ではなく、特定の期間だけ取り出したい場合：

1. ポップアップの **日付選択** で開始日と終了日を選ぶ
2. または **プリセットボタン**（1ヶ月・3ヶ月・6ヶ月・1年・全期間）をクリック

## 困ったときは（トラブルシューティング）

### 「認証トークンを取得できませんでした」と出る

**原因:** Google の設定がうまくいっていない可能性があります。

**やってみること:**
- [ ] ステップ1-5 で、テストユーザーに自分の Gmail を追加したか確認
- [ ] `manifest.json` のクライアント ID が、ステップ1-6 で作ったものと同じか確認
- [ ] 拡張機能を一度けして、もう一度入れ直してみる

### 「フォルダ移動には Google Drive API の有効化が必要です」と出る

**原因:** Google Drive API がオンになっていません。

**やってみること:**
- [ ] ステップ1-4 をやる（Google Drive API を有効にする）

### 「403 Forbidden」と出る

**原因:** API の権限が足りていません。

**やってみること:**
- [ ] ステップ1-3（Sheets API）と 1-4（Drive API）がオンになっているか確認
- [ ] ステップ1-5 のスコープ設定で、2つともチェックが入っているか確認

### 「このアプリは Google で確認されていません」が何度も出る

**これは正常です！** テストモード（個人利用）では毎回出ることがあります。
「詳細」→「安全ではないページに移動」で進んでください。

### 「抽出開始」ボタンが押せない（グレーのまま）

**原因:** 対応しているサイトの購入履歴ページを開いていないかもしれません。

**やってみること:**
- [ ] 上の表にある購入履歴ページを開いているか確認
- [ ] `chrome://extensions/` で拡張機能の更新ボタン（丸い矢印）を押してみる
- [ ] ページをもう一度読み込み直す（F5 キー）

## よくある質問

### Q. お金はかかりますか？
**A. かかりません。** Google Cloud Console の API は無料枠の範囲内で使えます（個人利用なら十分）。

### Q. 自分の購入情報が他の人に見られることはありますか？
**A. ありません。** データはあなたの Google アカウントのスプレッドシートに直接保存されます。開発者を含め、他の人がアクセスすることはありません。

### Q. スマホでも使えますか？
**A. 使えません。** Chrome 拡張機能はパソコン版の Chrome でのみ動作します。

### Q. 対応サイトを増やすことはできますか？
**A. 開発者にリクエストしてください。** 対応可能なサイトであれば追加できます。
